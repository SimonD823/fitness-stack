# Guide 07 — Daily Training Brief Email

Runs on the NAS every morning at 6:45 AM. Queries InfluxDB for yesterday's health data, calls LM Studio on Max for an AI coaching analysis, and emails an HTML daily brief to your Hotmail by 7:00 AM.

**Email contains:**
- AI coaching readiness assessment
- Today's specific training plan with targets
- Yesterday's sessions (cardio + strength detail)
- Sleep metrics (score, HRV, deep/REM, body battery)
- Activity summary (steps, distance, calories, resting HR)
- Nutrition summary (calories, macros from Cronometer)

---

## Step 1 — Outlook App Password

Microsoft requires an **App Password** for SMTP access when two-factor authentication (2FA) is enabled on your account. Your normal Hotmail password will not work for sending email from a container — Microsoft blocks it as a security measure.

**What is 2FA?** Two-factor authentication means logging in requires both your password and a second verification step (usually a code sent to your phone or generated by an authenticator app). If you haven't set this up, you'll need to enable it before you can create an App Password.

**Enable 2FA if not already active:**
1. Go to `https://account.microsoft.com/security`
2. Click **Two-step verification** and follow the setup wizard
3. You'll need the Microsoft Authenticator app or a phone number to receive codes

**Create the App Password:**
1. Go to `https://account.microsoft.com/security`
2. Click **Advanced security options**
3. Scroll down to **App passwords** and click **Create a new app password**
4. Microsoft generates a password automatically — you cannot choose the name or the password itself
5. **Copy it immediately** — it is only shown once and cannot be retrieved again
6. Paste it into the compose YAML as `EMAIL_PASS`

> **Important:** This is a long randomly-generated password (e.g. `xxxx xxxx xxxx xxxx`). Remove the spaces when pasting it into the YAML. If you lose it, just generate a new one — the old one becomes invalid.

## Step 2 — Create the directory in File Station

Open **File Station** → `Container`. Create:

```
Container / daily-brief
```

Copy `daily_brief.py` into `Container / daily-brief /` via Windows Explorer (`\\NAS_IP\Container\daily-brief`).

---

## Step 3 — Create the Docker Compose file

In File Station, create `docker-compose.yml` in `Container / daily-brief /`:

```yaml
version: "3.8"

services:
  daily-brief:
    image: python:3.12-slim
    container_name: daily-brief
    restart: unless-stopped
    environment:
      INFLUX_HOST: "NAS_IP"
      INFLUX_PORT: "8086"
      INFLUX_USER: "influxdb_user"
      INFLUX_PASS: "YOUR_INFLUX_PASSWORD"
      LM_STUDIO_URL: "http://MAX_IP:1234"
      LM_MODEL: "qwen/qwen3.6-27b"
      EMAIL_FROM: "your_email@example.com"
      EMAIL_PASS: "YOUR_APP_PASSWORD_HERE"
      EMAIL_TO: "your_email@example.com"
      SEND_HOUR: "6"
      SEND_MINUTE: "45"
      TRAINING_WEEK: "1"
      RUN_NOW: "false"
    volumes:
      - /share/Container/daily-brief/daily_brief.py:/app/daily_brief.py
    network_mode: host
    command: >
      bash -c "pip install --no-cache-dir requests influxdb --quiet &&
               python /app/daily_brief.py"
```

Replace `YOUR_APP_PASSWORD_HERE` with the App Password from Step 1.

> **TRAINING_WEEK** — update this manually each Monday to keep the AI context accurate. Week 1 = 19 May, Week 2 = 26 May, etc.

---

## Step 4 — Create the application in Container Station

Open **Container Station → Applications → Create**.

- **Application Name:** `daily-brief`
- Paste the compose YAML
- Click **Validate YAML → Create**

The container sleeps until 6:45 AM and sends the email then. Check the Logs tab to confirm it's running and see the next scheduled send time.

---

## Step 5 — Test immediately

To send a test email right now without waiting until 6:45 AM, temporarily change `RUN_NOW: "false"` to `RUN_NOW: "true"` in the compose YAML, delete and recreate the application, check your inbox, then set it back to `"false"`.

## Troubleshooting

**Email not arriving:**
- Check Container Station Logs for SMTP errors
- Verify the App Password is correct (not your main Hotmail password)
- Confirm `EMAIL_FROM` and `EMAIL_TO` are correct

**"Authentication failed" SMTP error:**
- Your account may need an App Password — see Step 1
- Some Hotmail accounts require enabling "Less secure app access" in security settings

**"LM Studio unreachable" in email:**
- Max must be on and logged in
- LM Studio must be running with the model loaded
- Check that port 1234 is open in Windows Firewall on Max

**AI response cut off:**
- Increase `max_tokens` in the script (currently 1500)
- Or reduce the amount of data sent in the prompt

**TRAINING_WEEK is wrong:**
- Edit the compose YAML in File Station → change the number → restart the container
